3 STEPS TO DEPLOY
================================================================================

STEP 1: Set Up Database (5 min)
─────────────────────────────────
Go to https://planetscale.com (or use AWS RDS/Google Cloud SQL)
1. Create account and new database
2. Name it: agrosmart
3. Copy connection string (host, user, password)
4. Run database/schema.sql in your MySQL client
   → This creates all tables needed

STEP 2: Deploy Flask Backend (5 min)
────────────────────────────────────
1. Install Vercel CLI: npm install -g vercel
2. In project root, run: vercel deploy --prod
3. Select root directory when prompted
4. Add these environment variables:
   DB_HOST=your-database-host
   DB_USER=your-database-user
   DB_PASSWORD=your-database-password
   DB_NAME=agrosmart
   SECRET_KEY=[generate random 32-char string]
   GOOGLE_MAPS_API_KEY=[your API key]

STEP 3: Deploy Other Services (10 min)
──────────────────────────────────────

Drone Backend:
   cd drone-control-system/backend
   vercel deploy --prod
   Set env var: FRONTEND_URL=[your frontend URL]

Drone Frontend:
   cd drone-control-system/frontend
   vercel deploy --prod
   Set env vars: VITE_BACKEND_URL, VITE_DRONE_SERVER_URL

================================================================================
TROUBLESHOOTING
================================================================================

"Could not import app.py"
   → wsgi.py is the entry point now, not app.py directly

"Database connection failed"
   → Check DB_HOST, DB_USER, DB_PASSWORD in Vercel env vars
   → Ensure database exists and schema is initialized
   → For PlanetScale: Check SSL is enabled

"Socket.io not connecting"
   → Verify FRONTEND_URL is set in drone backend env vars
   → Check browser console for connection error

"Module not found"
   → All dependencies are in requirements.txt
   → Rebuild with: vercel deploy --prod --build-env
